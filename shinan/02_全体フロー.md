# 全体フロー

## 本番フロー

```text
締切30〜20分前
  出走表取得
  公式rankingmotor取得
  直前情報取得
  モーター番長AI
  展示・進入チェックAI
  アレド判定
  高度分析AIで買い目候補

締切10〜5分前
  現在オッズ取得
  オッズ番人AI
  安すぎる券を削る
  妙味が残る券だけ最終候補

締切3分前
  オッズ再確認
  大きく崩れた券は削る
  最終購入
```

## 過去検証フロー

```text
予想前情報だけで input.json を作る
→ funayomi_pipeline.py で report.json を作る
→ report.json を保存
→ その後で result.json を取得
→ 的中/回収率を集計
→ 検証ログに戻す
```

## 判断の流れ

### 1. モーター番長AI

公式モーターを相対評価する。
モーターは材料であり、買い目の答えではない。

### 2. 展示・進入チェックAI

直前情報を読む。
展示タイム、スタート展示、進入ズレ、チルト、部品交換、水面を評価する。

### 3. 荒れ度診断AI / アレド

買う前のゲート。
星が少ないレース、読みにくいレース、モーター根拠だけで不安なレースを弾く。

### 4. 高度分析AI

通過したレースだけ買い目候補を作る。
基本は1号艇軸。
ただし、1号艇が弱い時は無理に買わない。

### 5. オッズ番人AI

本番専用。
AIが先に出した買い目候補だけを対象に、買う価値があるか判定する。

## 出力形式

各レースで最低限残すもの。

- 使用した公式URL
- モーター番長評価
- 展示・進入評価
- アレド判定
- 高度分析の買い目
- 見送り理由
- 結果
- 的中/不的中
- 払戻
- 学び
