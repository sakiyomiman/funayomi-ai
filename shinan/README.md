# 指南流（shinan）— プロンプト式フナヨミAI

フナヨミAIの**もう1つの流儀**です。`engine/` がPythonで数値を計算する「計算流」なのに対し、
こちらは**AIが設計書（このフォルダのMD群）を読んで、自分の判断で予想を組み立てる**「指南流」です。

Claude Code・Codex・その他の対話型AIエージェントであれば、どれでも動きます。

---

## 使い方

AIエージェント（Claude Code / Codex 等）でこのリポジトリを開いて、こう言うだけです：

```
shinan/ の設計書を読んで、蒲郡8Rを指南流で予想して
```

AIは以下の順で読み、分析フローに従って予想します。

| 順 | ファイル | 役割 |
|---|---|---|
| 1 | [00_全体設計.md](00_全体設計.md) | 思想・分析フロー・禁止事項（最初に読む1枚） |
| 2 | [01_憲法.md](01_憲法.md) | 絶対ルール（オッズで予想を作らない・負けデータを残す 等） |
| 3 | [AI_01_モーター番長.md](AI_01_モーター番長.md) | モーターだけを見る採点係 |
| 4 | [AI_02_展示進入チェック.md](AI_02_展示進入チェック.md) | 展示タイム・スタート展示・進入の直前気配 |
| 5 | [AI_03_荒れ度診断アレド.md](AI_03_荒れ度診断アレド.md) | **参加/見送りの門番**（ここで約半分を見送る） |
| 6 | [AI_04_高度分析.md](AI_04_高度分析.md) | 通過レースだけ買い目3〜6点に絞る |
| 7 | [AI_05_オッズ番人.md](AI_05_オッズ番人.md) | 最後に安すぎる券を削るだけ（予想には関与しない） |

運用の詳細は [02_全体フロー.md](02_全体フロー.md)・[03_データ漏洩防止ルール.md](03_データ漏洩防止ルール.md)・[04_検証蓄積ループ.md](04_検証蓄積ループ.md) を参照。

## 計算流との違い（どっちを使う？）

同じ50レース・同じ資金ルール（1R 1,000円均等割）で2つの流儀を対決させた実測です（2026-07-06・全会場ランダム50R）。

| | 指南流（このフォルダ） | 計算流（engine/） |
|---|---|---|
| 参加 | 22R / 見送り28R | 17R / 見送り33R |
| 的中率 | **32%** | 6% |
| 回収率 | **86.0%** | 20.1% |
| 見送り正解率（荒れ回避） | 57% | **67%** |

- **指南流の強み**：筋読み（2着・3着の相手選び）と「1-2-5 / 1-2-6」のような保険買い目。的中率が高い。
- **計算流の強み**：見送りの目利きと、走るたびに学習して育つ brain（長期記憶）。再現性が完全。
- **正直な注記**：この50Rでは両者ともマイナスでした。指南流の86%も高配当1本込みの数字です（除くと48%）。
  50レースはまだ統計的に荒く、良い結果だけ切り抜かないのが本プロジェクトの理念です。
  計算流は「勝てる場」に絞った運用（[docs/venue_theory.md](../docs/venue_theory.md)）が本来の使い方で、全場このまま買う設計ではありません。

**おすすめ**：まず計算流でレポートを作り、参加判定が出たレースだけ指南流でセカンドオピニオンを取る使い方が、両者の強みを活かせます（どちらかが見送りなら見送り）。
