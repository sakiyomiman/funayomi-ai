# データ漏洩防止ルール

## なぜ重要か

過去レース検証で、結果・最終オッズ・払戻・人気を予想前に使うと、後出しで勝っているだけになる。

サキヨミAIはこれを絶対に避ける。

## ファイル分離

採用する分離。

```text
inputs/
  race_input.json        # 予想前情報だけ
predictions/
  race_report.json       # AIの予想
results/
  race_result.json       # 答え合わせ専用
summaries/
  summary.md             # 集計
```

## input.jsonに入れてよいもの

- 出走表URL
- 直前情報URL
- 公式rankingmotor URL
- 選手情報
- モーター情報
- 展示タイム
- スタート展示
- チルト
- 部品交換
- 気象、水面

## input.jsonに入れてはいけないもの

- `actual_result`
- `result`
- `payout`
- `popularity`
- `odds_final`
- `source_urls.result`

## オッズの扱い

### 過去検証

オッズは使わない。
当時の途中オッズを保存していない限り、期待値検証はしない。

### 本番

現在オッズは使ってよい。
ただし、買い目を作るためではなく、買う/削るためにだけ使う。

## チェックリスト

予想前に確認する。

- inputに結果が入っていないか
- inputに払戻が入っていないか
- inputに最終人気が入っていないか
- inputに締切時オッズが入っていないか
- アレド落ちレースを買っていないか
