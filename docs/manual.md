# フナヨミAI 使い方マニュアル

はじめての方向けに、導入から「脳を育てる」までを順番に説明します。

---

## 0. 必要なもの

- **Python 3.9以上**（Macなら最初から入っています。ターミナルで `python3 --version` と打って確認できます）
- インターネット接続
- 以上です。追加のライブラリのインストールは不要です。

## 1. 導入（最初の1回だけ）

```bash
git clone https://github.com/sakiyomiman/funayomi-ai
cd sakiyomi-ai
```

gitが分からない場合は、GitHubのページ右上「Code」→「Download ZIP」でダウンロードして解凍しても同じです。

## 2. レースを予想する

### 2-1. 場コードを調べる

会場は2桁の番号で指定します。

| 01 桐生 | 02 戸田 | 03 江戸川 | 04 平和島 | 05 多摩川 | 06 浜名湖 |
| 07 蒲郡 | 08 常滑 | 09 津 | 10 三国 | 11 びわこ | 12 住之江 |
| 13 尼崎 | 14 鳴門 | 15 丸亀 | 16 児島 | 17 宮島 | 18 徳山 |
| 19 下関 | 20 若松 | 21 芦屋 | 22 福岡 | 23 唐津 | 24 大村 |

### 2-2. 実行する（発走30〜60分前に）

蒲郡8Rを予想する例です。

```bash
python3 engine/fetch.py --jcd 07 --rno 8
python3 engine/engine.py
python3 engine/gpts.py --inject
python3 engine/build.py
```

`races/` フォルダにHTMLレポートができるので、ダブルクリックでブラウザで開いてください。

### 2-3. 「直前情報がまだ」と言われたら

フナヨミAIは、展示航走のデータ（発走の30〜60分前に公開）が出てから予想する設計です。
これより早い時間に実行すると「まだ公開されていません」と止まります。**これは故障ではなく、精度のための仕様です。**
時間をおいて再実行してください。

### 2-4. 「見送り」と言われたら

荒れ度AIが「このレースは危ない」と判断すると、買い目を出さずに「見送り」と答えます。
検証では全レースの約半分が見送りでした。**見送りが多いのは正常です。**
むしろ全レースに買い目を出すAIの方を疑ってください。

## 3. 脳を育てる（1日の終わりに）

その日のレースが終わったら、2つのコマンドを実行します。

```bash
python3 engine/collect.py --hd 20260705 --jcd auto   # 日付は当日をYYYYMMDDで
python3 engine/learn.py
```

- `collect.py` … 今日の全レース結果を `data/races/` に記録します（エピソード記憶）
- `learn.py` … 記録をもとに反省会をします。学んだことは検証テストに合格した場合だけ `brain/params.json` に書き込まれ、`brain/lessons.md` に記録が残ります

これを続けるほど、「参加する場・見送る場」の判断があなたの使う環境に合わせて更新されていきます。

## 4. 過去レースで検証する（バックテスト）

エンジンの実力を自分で確かめられます。

```bash
# 過去の日付・会場を指定して収集
python3 engine/collect.py --hd 20260610 --jcd 07

# 貯めたデータ全部で成績を出す（1秒で終わります）
python3 engine/replay.py

# 会場や日付で絞ることもできます
python3 engine/replay.py --jcd 07 --hd 202606
```

回収率・的中率・見送り精度・会場別成績が表示されます。
**私たちの公表数字が信用できるかどうか、ぜひ自分の手で確認してください。** それがこのプロジェクトの理念です。

## 5. よくある質問

**Q. 本当に勝てますか？**
A. 全レースを買えば負けます（実測で回収率84%）。検証で「勝てる場」に絞った場合のみ、110%前後の実績があります。ただし過去の実績は未来を保証しません。

**Q. どの場がおすすめですか？**
A. 検証時点では蒲郡・尼崎・浜名湖です。理由は [venue_theory.md](venue_theory.md) をご覧ください。

**Q. エラーで止まりました**
A. Issueに「質問」と書いて、実行したコマンドとエラーメッセージを貼ってください。

**Q. 予想の中身（ロジック）を変えたい**
A. `engine/` のコードはすべて公開しています。変更したら `replay.py` で変更前後の回収率を比べてください。感覚でいじると、だいたい悪化します（開発者も何度もやりました）。
