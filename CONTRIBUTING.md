# Contributing — フナヨミAIへの参加方法

参加ありがとうございます。このプロジェクトは、コードが書ける人だけのものではありません。
競艇が好きで、検証に興味がある人なら誰でも貢献できます。

## 一番ほしい貢献：検証データと場の知見

### 1. 検証ログの共有

あなたの地元の場でフナヨミAIを回した結果を教えてください。書式は自由ですが、以下があると助かります。

```
- 期間と場: 2026年7月・徳山
- レース数: 参加◯R / 見送り◯R
- 結果: 的中◯回 / 回収率◯%（1レース1,000円換算）
- 気づいたこと: 「★1判定なのにインが飛ぶ日は、だいたい向かい風だった」など
```

Issueに「検証ログ」のタイトルで投稿してください。`data/experiments/` へのプルリクエストでも歓迎です。
**負けた検証ログほど価値があります。** どこで負けるかが分かれば、AIは賢くなれます。

### 2. 場の知見

「この場は追い風の日はセンターが伸びる」「冬場は水面が硬くてまくりが決まらない」——
通っている人にしか分からない知識は、データからは見えません。Issueで気軽に教えてください。
検証で裏が取れたものは、出典を明記して `docs/venue_theory.md` に反映します。

### 3. コード・ドキュメントの改善

もちろん歓迎です。特にロードマップにある以下は助けが要ります。

- 未対応7場（戸田・津・丸亀・児島・宮島・徳山・唐津）の会場データ取得（`engine/motorbank.py` のパーサ追加）
- 欠場レース（5艇立て）対応
- ドキュメントの分かりやすさ改善

## プルリクエストのルール（最小限です）

1. **予想ロジック（engine/）の変更は、検証結果を添えてください。** `engine/replay.py` で変更前後の回収率を比較できます。「なんとなく良くなりそう」だけの変更は、過去に何度も回収率を悪化させてきました（検証記録参照）。
2. データ取得系の変更は、**公式サイトへのアクセス間隔に配慮**してください。
3. 大きな変更は、先にIssueで相談してもらえるとお互いに無駄がありません。

## はじめてのOSS参加の方へ

はじめてのIssue・はじめてのPR、大歓迎です。書式の間違いなどは気にしなくて大丈夫です。
分からないことがあれば、Issueに「質問」と書いて投げてください。

## 行動規範

[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) をご覧ください。
要約すると「お互いに敬意を持って、数字で建設的に話しましょう」です。
