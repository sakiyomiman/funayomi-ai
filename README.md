## 理念

フナヨミAIは「競艇予想AI研究所」のプロジェクトです。大事にしていることは3つです。

1. **当てる詐欺をしない。** 良い結果だけを切り抜いた宣伝はしません。負けた検証もそのまま公開します（[data/experiments/](data/experiments/)）。
2. **数字で語る。** 実際のレースデータで検証できたことだけを書きます。
3. **参加しない勇気を持つ。** 競艇は売上の約25%が最初から控除されるため、全レース買うと必ず負ける構造になっています。だから「買わない」という判断を、予想の一部として一番大事にしています。

## これは何？

競艇（ボートレース）の予想を、4つのAIが順番に処理して組み立てるエンジンです。

```
モーターAI → 荒れ度AI → 本気分析AI → オッズAI
```

| AI | 役割 | 実装ファイル |
|---|---|---|
| **モーターAI** | 6艇のモーター（エンジン）の強さを、会場の公式データから採点する | [engine/motorbank.py](engine/motorbank.py) |
| **荒れ度AI** | そのレースが荒れそうかを判定する。荒れそうなら**そのレースには参加しない** | [engine/engine.py](engine/engine.py) |
| **本気分析AI** | 参加すると決めたレースだけ、展開を読んで買い目を3〜6点に絞る | [engine/gpts.py](engine/gpts.py) |
| **オッズAI** | 最後のチェック役。うまみのない券を削るだけで、予想内容には関与しない | [engine/gatekeeper.py](engine/gatekeeper.py) |

この4つの後ろに **brain（脳）** があります。毎晩その日のレース結果を振り返って学習する長期記憶で、配布時点では開発チームが677レース分検証して育てた状態（[brain/](brain/)）が入っています。ここから先は、あなたの使用記録で、あなた用に育っていきます。

### 2つの流儀 — 計算流と指南流

フナヨミAIには、同じ憲法（オッズで予想を作らない・見送りも予想・負けも公開）を共有する**2つの実装**があります。

| | 計算流（[engine/](engine/)） | 指南流（[shinan/](shinan/)） |
|---|---|---|
| 動く仕組み | Pythonが数値を計算（決定論的） | AIが設計書を読んで判断（プロンプト式） |
| 必要なもの | Python 3.9+ | 対話型AIエージェント（Claude Code / Codex 等） |
| 強み | 見送りの目利き・学習して育つbrain・再現性 | 筋読み（2着3着の相手選び）・保険買い目・的中率 |
| 検証 | 677レース（walk-forward学習つき） | 検証ログを [shinan/kensho/](shinan/kensho/) に公開 |

同一50レースで対決させた実測比較（負けた側の数字も含めて）は [shinan/README.md](shinan/README.md) に載せています。
おすすめは併用です：計算流でレポートを作り、参加判定のレースだけ指南流でセカンドオピニオンを取る（どちらかが見送りなら見送り）。

### 設計上のこだわり

- **オッズで予想を作らない。** オッズから買い目を逆算する方式も最初に試しましたが、人気に引きずられるだけになり全敗しました（下の[検証記録](#検証記録失敗も含めて)に記載）。オッズは最後の確認にだけ使います。
- **思い込みを学習しない。** 数日分のデータで学んだことは、たいてい偶然です。フナヨミAIは「その知恵が、学習に使っていない日のレースでも通用するか」を毎回テストし、合格したものだけを記憶します（[engine/learn.py](engine/learn.py)）。

## リポジトリの歩き方（どのファイルが何を担い、なぜそうなっているか）

「なぜこの予想になるのか」「なぜ脳がこう育ったのか」を追えるように、ファイル同士をリンクで繋いでいます。

### 予想を作る流れ（上から順に実行される）

| ファイル | 役割 | なぜ必要か |
|---|---|---|
| [engine/fetch.py](engine/fetch.py) | boatrace.jp から実測データを取得（出走表・展示・オッズ・気象） | ここで直前情報が取れなければ何も始まらない。展示前は `exit 2` で止める＝精度のためのゲート |
| [engine/motorbank.py](engine/motorbank.py) | モーターAI。会場公式のモーター成績を採点 | 「その日そのモーターが動くか」は展示だけでは読めない。公式の積み上げで補正する |
| [engine/engine.py](engine/engine.py) | 分析計算の司令塔。コース基礎値×選手力×モーター×荒れ度×期待値ゲートを統合 | **数値計算はすべてここ**。AI（人間や言語モデル）は数値を作らず、ここが出した数字を使うだけ |
| [engine/gpts.py](engine/gpts.py) | GPTs式3体チェーン（bancho→honki→odds）を注入し、買い目・判定を確定 | 素の期待値ゲートは万券ばかり買って回収率0%だった。実運用で育てた3体チェーンで上書きする |
| [engine/gatekeeper.py](engine/gatekeeper.py) | オッズAI／番人。うまみの無い券を削るだけ | **買い目の追加・変更は絶対にしない**（オッズで予想を作らない＝本プロジェクトの鉄則） |
| [engine/build.py](engine/build.py) | 予想レポートHTMLを `races/` に生成 | 人が読む最終成果物。[template/report.html](template/report.html) を使う |

補助データ：[engine/course_base.json](engine/course_base.json)（24場のコース基礎値）

### 脳を育てる流れ（1日の終わりに回す）

| ファイル | 役割 | なぜ必要か |
|---|---|---|
| [engine/collect.py](engine/collect.py) | その日のレース結果を収集（エピソード記憶づくり） | 「予想が当たったか」の生データが無ければ学習も検証もできない |
| [engine/learn.py](engine/learn.py) | 学習。walk-forward検証で過学習を弾く | 数日分に合わせ込むと別日で崩壊する（実際に104%→68%で崩壊した）。別日で通用したものだけ記憶する |
| [engine/replay.py](engine/replay.py) | バックテスト。貯めたエピソードで回収率を再現 | 公表数字を**自分の手で**検証できる。理念の「数字で語る」の実装 |
| [engine/result.py](engine/result.py) / [engine/verify.py](engine/verify.py) | 確定結果の照合・検証ユーティリティ | 収集・学習が正しく動いているかを裏取りする |

## 検証記録（失敗も含めて）

最初の失敗から順に載せます。

| 段階 | 方式 | 検証数 | 回収率 | 学んだこと |
|---|---|---|---|---|
| v0 | オッズから期待値の高い券を選ぶ | 46レース | **0%** | 大穴ばかり選ぶようになり全敗。オッズで予想を作ってはいけない |
| v1 | 4体AIの分業方式に変更 | 46レース | 91% | 展開を読んで数点に絞る方式が正しかった |
| v2 | 「参加しない」判定を厳格化 | 234レース | 84% | これが素の実力。基準点とした |
| — | 調整の結果104%に到達 | 別日155レース | **68%** | 別の日で検証したら崩壊。数日分への合わせ込み（過学習）だった |
| v3 | 学習内容を別日データで毎回テストする仕組みを導入 | 448レース | — | 思い込みを記憶しない構造が完成 |

※回収率＝投じた金額に対して戻ってきた金額の割合。100%を超えればプラスです。

### 現在の実力（2026年6〜7月・677レース収集）

全会場をそのまま買うと**回収率は約84%**です。これが正直な素の数字です。
ただし、学習した「勝てる場」に絞ると結果が変わります。

| 会場 | 検証数 | 的中率 | 回収率 |
|---|---|---|---|
| 尼崎 | 81レース | 33% | **115%** |
| 浜名湖 | 53レース | 37% | **112%** |
| 蒲郡 | 110レース | 31% | **106%** |
| （参考）住之江 | 31レース | 21% | 67% |

3場の合計は132レース参加、投資116,900円 → 回収128,810円（+11,910円）。
「見送り」判定の的中率は約7割で、34,790円の大波乱レースを事前に回避した例もあります。
詳しい理由は [docs/venue_theory.md](docs/venue_theory.md) をご覧ください。

## 使い方（3ステップ）

Python 3.9以上が入っていれば動きます。追加のインストールは不要です。

```bash
git clone https://github.com/sakiyomiman/funayomi-ai
cd sakiyomi-ai
python3 engine/fetch.py --jcd 07 --rno 8   # 蒲郡8Rのデータを取得（発走30〜60分前に実行）
python3 engine/engine.py && python3 engine/gpts.py --inject && python3 engine/build.py
```

`races/` フォルダに予想レポート（HTML）が生成されます。
くわしい手順は [docs/manual.md](docs/manual.md) をご覧ください。

### 脳を育てる（1日の終わりに2コマンド）

```bash
python3 engine/collect.py --hd 20260705 --jcd auto   # 今日のレース結果を記憶に追加
python3 engine/learn.py                              # 学習（別日データでの検証つき）
```

学習した内容は [brain/lessons.md](brain/lessons.md) に記録として残ります。
